[ 1. 현재 게임 구조 요약 ]

  핵심 루프
    WaitingRoom(스킬 구매/준비) → InGame(채굴+방어) → WaitingRoom 귀환

  주요 시스템
    - Marching Cubes 복셀 지형 (Air/Protein/Iron/Calcium/GeneticEssence)
    - 적(면역 세포) 웨이브 소환 → Matriarch 방어
    - 아이템 인벤토리·크래프팅
    - 개인 스킬 트리 (이동속도·점프력·체력) — raw_genetic_essence로 구매
    - 멀티플레이어 협동 (NGO)


================================================================================
[ 2. 현재 게임의 강점 ]

  (1) 복셀 지형 채굴 + 방어의 결합
      - "채굴로 자원을 모아 방어를 강화" 루프가 명확하다.
      - 지형 자체가 전술 요소가 될 수 있다 (터널 파기, 참호 구축).

  (2) 자원 경도 차등
      - 채굴 난이도(경도)가 아이템 희귀도와 연동 → 위험 대비 보상 구조.

  (3) 개인 스킬 트리
      - 플레이어마다 다른 빌드를 선택할 수 있어 역할 분담 가능성 있음.

  (4) 웨이브 스케일링
      - 시간이 지날수록 마리수·스탯이 오르므로 긴장감이 자연스럽게 증가.

[ 3. 재미를 떨어뜨리는 문제점 분석 ]

  (A) 목표 불명확성
      - "Matriarch를 지킨다"는 목표 외에 중간 목표가 없다.
      - 플레이어가 지금 무엇을 해야 하는지 모르는 시간이 발생.
      - 적이 언제 오는지 UI로 알리지 않아 채굴 타이밍을 잡기 어렵다.

  (B) 협동 동기 부족
      - 스킬 트리가 개인 독립적이라 협동 플레이 인센티브가 약하다.
      - "같이 하면 더 유리하다"는 느낌이 아직 없다.

  (C) 전투 피드백 빈약
      - 적을 때렸을 때 히트리액션·사운드·파티클이 없으면 타격감 없음.
      - Matriarch의 HP가 시각적으로 잘 보이지 않으면 위기감을 못 느낌.

  (D) 아이템·크래프팅 가치 희석
      - 아이템이 많더라도 스킬 트리 보너스가 너무 크면
        아이템을 만들 동기가 사라진다.
      - 크래프팅 레시피가 단조로우면 후반부에 할 일이 없어진다.

  (E) 지형 채굴의 목적성 부족
      - 현재는 "광물 = 아이템 재료" 뿐이다.
      - 지형을 파는 행위 자체에 전술적 가치가 없으면 채굴이 심심해진다.

  (F) 웨이브 간격 사이의 공백
      - 웨이브와 웨이브 사이 시간에 "아무것도 안 해도 된다"는
        느낌을 주면 플레이어가 집중을 잃는다.

[ 4. 재미 향상을 위한 추가 요소 제안 ]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  [CATEGORY A] 핵심 게임플레이 루프 강화
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  A-1. 웨이브 카운트다운 HUD
       - 다음 웨이브까지 남은 시간을 화면에 표시.
       - 플레이어가 "지금 채굴할 시간" vs "방어 준비 시간"을 판단할 수 있다.
       - 구현 난이도: 낮음 (EnemySpawner의 _timer를 NetworkVariable로 노출)

  A-2. Matriarch HP 월드 UI (체력바)
       - Matriarch 머리 위 또는 맵 한쪽에 대형 HP바 표시.
       - "Matriarch가 죽으면 게임 오버"라는 긴장감이 시각화됨.
       - 구현 난이도: 낮음

  A-3. 웨이브 클리어 보상
       - 웨이브를 완전히 소탕하면 추가 skill point 또는 희귀 아이템 지급.
       - 채굴뿐 아니라 전투도 적극적으로 참여할 동기가 생긴다.
       - 구현: EnemySpawner에서 활성 적 수를 추적, 0이 되면 보상 발행

  A-4. 긴급 이벤트 (엘리트 적)
       - 일정 웨이브마다 일반 적보다 훨씬 강하고 특수 능력을 가진
         "엘리트 면역세포"가 등장.
       - 예: 지형을 빠르게 파괴하는 적, Matriarch를 직선으로 돌진하는 적.
       - 구현: EnemySpawner에 eliteSpawnInterval 추가, 별도 프리팹 소환

  [CATEGORY B] 협동 플레이 강화
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  B-1. 역할 분담 시스템 (Role)
       - 게임 시작 전 WaitingRoom에서 역할 선택:
           * 채굴자(Miner)    : 채굴 속도 +30%, 드롭 수량 +20%
           * 전투원(Fighter)  : 공격력 +20%, 피격 시 넉백 저항
           * 지원가(Support)  : 아이템 사용 시 주변 플레이어에도 효과 적용
       - 역할은 스킬 트리와 독립적으로 패시브 효과를 줌.
       - 구현: RoleDataSO + PlayerRole enum, WaitingRoom UI에 선택 버튼

  B-2. 공유 방어 구조물 설치
       - 채굴한 자원으로 "방어 터렛", "장벽 블록", "회복 스테이션"을
         지형 위에 설치 가능.
       - 한 플레이어가 채굴하고 다른 플레이어가 방어선을 구축하는
         자연스러운 역할 분담 유도.
       - 구현: PlaceableItemSO 확장, MarchingChunk에 구조물 레이어 추가

  B-3. 자원 공유 핫드롭
       - 플레이어가 아이템을 세계에 드롭하면 다른 플레이어가 자동 흡수 가능.
       - 채굴 전담 플레이어가 전투 전담 플레이어에게 재료를 보급하는 구조.
       - 구현: 이미 드롭 아이템 자동 흡수가 있으므로 UI만 추가

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  [CATEGORY C] 전투 다양성·깊이 향상
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  C-1. 적 타입 다양화
       현재 적은 단일 타입으로 보임. 아래 타입을 추가하면 전술이 다양해진다:
       ┌─────────────────┬───────────────────────────────────────────────┐
       │ 타입명           │ 특성                                         │
       ├─────────────────┼───────────────────────────────────────────────┤
       │ 돌격 세포        │ 빠른 속도, 낮은 HP — 현재 기본 타입          │
       │ 탱커 세포        │ 느린 속도, 높은 HP, 플레이어를 밀쳐냄        │
       │ 폭발 세포        │ Matriarch 도달 시 광역 데미지 폭발            │
       │ 산 분비 세포     │ 주변 복셀을 빠르게 녹임 (지형 파괴)          │
       │ 치유 세포        │ 주변 적을 회복 — 우선 제거 필요              │
       └─────────────────┴───────────────────────────────────────────────┘
       - 구현: EnemyDataSO에 enemyType 필드 추가, EnemyController 분기 처리

  C-2. 원거리 무기 추가
       - 현재 MeleeWeaponSO만 존재. 원거리 무기(발사체) 추가 시
         전투 포지셔닝이 의미 있어진다.
       - 구현: RangedWeaponSO : ItemDataSO, ProjectileController

  C-3. 상태이상 시스템
       - 적에게 상태이상(느려짐·독·빙결)을 적용하는 아이템·스킬.
       - 지원가 역할과 시너지 → 협동 전투의 깊이를 높인다.

  C-4. 지형 전술 (복셀 활용)
       - 적 경로에 구멍을 파서 적을 떨어뜨리기.
       - 높은 위치에서 적을 공격하는 고지 유리함.
       - Calcium 벽을 세워 임시 방어선 구축.
       - 이를 위해 복셀 블록을 "설치"할 수 있는 도구 필요.
       - 구현: ModifyTerrain 반대 방향 (밀도 증가) 아이템

  [CATEGORY D] 탐험·채굴 재미 강화
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  D-1. 광맥 시스템 (Ore Vein)
       - 특정 깊이에 고농도 광맥이 생성됨.
       - "깊이 들어갈수록 희귀 자원, 하지만 돌아오는 데 시간이 걸린다"
         → 위험 대비 보상 의사결정 발생.
       - 구현: WorldGenerator에 depth 기반 VoxelType 확률 조정

  D-2. 숨겨진 동굴 (Hidden Cavity)
       - 지하에 미리 생성된 빈 공간, 내부에 희귀 아이템 상자 배치.
       - 탐험 동기 부여.

  D-3. 자원 고갈 & 재생
       - 채굴한 구역은 시간이 지나면 일부 복셀이 재생(GeneticEssence 제외).
       - 자원을 독점하지 않고 팀원과 나눠 채굴하게 유도.

  D-4. 채굴 깊이 보너스
       - Matriarch로부터 일정 거리 이상 멀리, 또는 일정 깊이 이하에서
         채굴 시 드롭 수량 보너스.
       - 구현: PlayerController.Mine()에서 거리 계산 후 배율 적용

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  [CATEGORY E] 성장·진행 만족감
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  E-1. 스킬 트리 확장 (브랜치 다양화)
       - 현재: 이동속도/점프/HP 3트리
       - 추가 제안:
           * 채굴 효율 트리: 채굴 속도, 드롭 보너스, 채굴 범위
           * 전투 트리: 공격속도, 크리티컬 확률, 연속공격
           * 지원 트리: 아이템 쿨타임 감소, 팀 버프 범위, 부활 시간 감소
       - 이 중 채굴/전투 트리는 역할 분담 선택에 직접적 동기 부여

  E-2. 런 내 영구 업그레이드 vs 라운드 내 임시 강화
       - 현재 스킬 트리는 WaitingRoom에서만 사용 가능 (런 지속 효과).
       - "게임 중 드랍되는 임시 버프 아이템"을 추가해 라운드 내 성장감 부여.
       - 예: "이번 라운드 동안 이동속도 +2" 효과의 소비 아이템

  E-3. 웨이브 클리어 기록 + 하이스코어
       - 최대 생존 웨이브 수, 처치 수, 총 채굴량을 WaitingRoom에 표시.
       - 플레이어끼리 기록 경쟁 유도.
       - 구현: GameManager에 통계 NetworkVariable 추가

  E-4. Matriarch 성장
       - Matriarch에게 자원을 헌납하면 Matriarch 자체가 강해짐:
           * HP 증가, 자동 방어 사격, 범위 회복 오라 등
       - 팀 전체가 협력해 Matriarch를 키우는 공동 목표 생성.

  [CATEGORY F] 분위기·몰입감
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  F-1. 비주얼 피드백 강화
       - 적 피격 시 히트마크 파티클 + 짧은 화면 진동(카메라 쉐이크).
       - Matriarch HP 30% 이하 시 화면 주변부 빨간 빛(Vignette).
       - 채굴 시 광물별 다른 색상 파티클 (Calcium=흰색, Iron=주황, GE=보라).

  F-2. 사운드 디자인
       - 각 적 타입별 다른 효과음.
       - 웨이브 시작 직전 "심박수" 사운드로 긴장감 고조.
       - 채굴음이 광물 경도에 따라 다름 (부드러운 Protein vs 묵직한 Calcium).

  F-3. 세계관 강화 (생체 테마)
       - 게임 세계가 생명체 내부라는 설정을 지형과 적 디자인에 반영.
       - 혈관처럼 생긴 터널, 세포막처럼 생긴 지형 표면 등.
       - 스킬 트리 이름을 생물학 용어로 표현:

[ 6. 핵심 게임 정체성 제언 ]

  Bio-Breach의 가장 독특한 요소는
  "살아있는 신체 내부를 무대로 한 복셀 채굴 + 웨이브 디펜스" 조합이다.

  이 정체성을 살리려면 세 가지 경험이 항상 연결되어야 한다:

      [채굴]  →  [자원으로 성장]  →  [방어에 활용]
         ↑                                  |
         └──────── [위험 = 보상] ───────────┘

  모든 추가 요소는 이 삼각형 루프를 강화하는 방향으로 설계할 것을 권장한다.
  (예: 새로운 적 → 더 강한 채굴 도구 필요 → 깊은 곳의 희귀 광물 채굴 촉진)
